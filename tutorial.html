Passo a Passo de instalação e configuração do git no windows

Por Maria Eduarda Ribeiro Alves

1.Instalação do git no windows

Há duas formas de instalar o git; Pelo terminal e pelo site oficial. O mais recomendado é pelo site oficial do git:

Link do git: Downloads do Git

Ao entrar no link abrirá a pagina inicial do git

Escolha a opção de instalar de acordo com o seu windows, em seguida pode abrir o programa normalmente.

2.Configuração pelo Git Bash

Criado a pasta que você subir para o git, abrá esse arquivo usando o atalho do git bash clicando no lado direito do mouse e escolhende a opção "git bash". Com o terminal ja aberto você irá usar os comandos de configuração global.

Comando:

git config --global user.nome "seu usuário do git"

git config --global user.email "seu email"

Esse comando faz com que todas as configurção sejam aplicadas, globalmente, em todos os respositórios desse usário "user.name"

3.Iniciar e Adicionar

Depois de ser configurado o git você irá iniciar com o comando:git init no terminal do bash

Ele cria um repositorio dentro do seu direito e cria uma pasta oculta ".git" no seu projeto

Após esse comando ter rodado com sucesso, você ira adicionar o seu documento com o comando git add "nome do seu documento"

Esse comando prepara o arquivo para sofrer alterações na área de preparação

Depois de adicionar o arquivo, será hora de colocar o seu commit ele salvará alterações no histórico do repositório. Use o comando git commit -m "escolha um nome"

e logo após você irá adicionar o link do seu repositório, usando o comando git remote add origin "link do repositório"

4.Git Branch

Uma branch é uma linha independente de desenvolvimento. Permite trabalhar em novas funcionalidades ou corrigir bugs sem afetar o código principal ( main ou master). Na hora da configuração você terá que mudar seu branch de master para main. use o coódigo git branch -m master main. para verificar se mudou é so usar o comando git branch

5.Enviar o arquivo para github

Hora de subir esse arquivo para o github de vez! usando o comando git push -u origin main o arquivo irá direto para o git. Mas, antes de dar esse comando terá que terminar a configuração usando o token do seu repositório!

5.1 Token de repositório

Para pegar o token você deve entar no github e acessar sua conta. depois de acessar sua conta você irá para o perfil e configurações.

irá descer ate o final onde terá a opção "configuração do desevolvedor"

Caso tenha acontecido não se desespere! apenas um comando pode ajudar! Pode apenas ser que usa url nãp tenha sido aceita ou seja vc precisa permetir, basta usar o comando git config --global credential.helper store e apos dar "enter", dê, novamente, git push -u origin main
